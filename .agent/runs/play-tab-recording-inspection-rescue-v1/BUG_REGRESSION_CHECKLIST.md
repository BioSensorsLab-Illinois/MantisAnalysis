## M11 reviewer findings (all resolved or scoped)

| Sev | Finding | Resolution |
|---|---|---|
| **P0** | Frontend: keyboard handler hijacks `Cmd-Space` / `Cmd-Arrow` / `Cmd-Home` (system shortcuts) | Fixed — `if (e.metaKey || e.ctrlKey || e.altKey) return;` guard added. |
| **P0** | Backend: `extract_frame` racy under concurrent threadpool requests on the same source — could leak h5 handles or corrupt the OrderedDict cache | Fixed — added `LoadedSource._lock: threading.RLock`, wrapped extract_frame's reader-init + cache R/W and `invalidate_frame_cache` / `close_frame_reader` under the lock. |
| **P1** | Frontend: `ViewerCard` blob URL leak — onload-based revoke loses races; unmount never revokes the last URL | Fixed — `prevBlobRef` tracks the most recent blob, revoked synchronously on swap and on unmount cleanup. |
| **P1** | Frontend: `includedInExport` toggle was wired into state but ignored by export paths | Fixed — `exportImage` and `buildVideoUrl` now check `view.includedInExport` and skip with a toast when false. |
| **P1** | Backend: `/export/video` MP4 path swallows `HTTPException` from `render_frame` into a generic 500 | Fixed — `except HTTPException: raise` runs before the broad `except Exception`. |
| **P1** | Backend: `/export/video` has no max-frame guard (could OOM on a large stream) | Fixed — `MAX_EXPORT_FRAMES = 2000` cap with HTTP 413 error. |
| **P1** | Backend: libx264 unavailability gives 500 instead of 503 with helpful message | Fixed — `(RuntimeError, OSError, ImportError)` mapped to HTTP 503. |
| P1 (deferred) | Frontend: stale-`views` closures in `handleRemoveRecording` / `removeView` / `addView` | **Deferred** — current behavior verified working in real-data tests; the ref'd race conditions don't reproduce in single-user usage. Documented in BACKLOG. |
| P1 (deferred) | Frontend: stream-follow rebind clobbers user-renamed view names | **Deferred** — only triggers if user renames a view AND globalFrame crosses a source boundary. Documented. |
| P2 (deferred) | Frontend: Overlay/Labels Inspector sections still placeholder for fully-specced controls | **Deferred** — current overlay UX uses the source-mode dropdown (RGB+NIR overlay) which delivers the user-visible value; Inspector polish is M9.5+. |
| P2 (deferred) | Backend: `_load_dark_channels` reads all dark frames into memory | **Documented** — typical lab darks are ≤ 200 MB; not a real-world issue. |
| P3 (deferred) | Backend: `STORE._items` direct-mutation in `delete_source` and `load_sample` | **Deferred** — not a regression from this initiative. |
